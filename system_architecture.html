● PHASE 4 — SMART CONTRACT STORES ALL REPORTS
📜

WebsiteMonitor.sol — Smart Contract (Solidity)

Deployed on Geth private network · stores every report immutably · cannot be edited or deleted
STEP 3
submitReport()
Called by each node. Stores url, status, response_time, ssl_valid, content_hash, node address.
updateReputation()
Called by ML pipeline. Updates node's reputation score and ALLOW/WARN/QUARANTINE/SLASH action.
recordConsensus()
Stores final website verdict — UP/DOWN, confidence %, votes count, avg response time.
slashNode()
Called when ML detects malicious node. Sets reputation to 0, marks as slashed, emits event.
getWebsiteHistory()
Returns full immutable history of all reports for a URL. Dashboard reads this to show timeline.
getNodeReputation()
Returns current reputation score and action status for any node address. Dashboard displays this.

● PHASE 7 — REPUTATION ENGINE + ACTION ASSIGNMENT
⚖️

Reputation Engine — Bayesian Update + EWMA Smoothing

Updates trust score per node every epoch · assigns final action
STEP 6
Bayesian Odds Update
prior_odds = rep / (1 - rep)
likelihood = (1 - mal_prob) / mal_prob
posterior = prior × likelihood
new_rep = posterior / (1 + posterior)
EWMA Smoothing (α = 0.9)
smoothed = α × old_rep + (1-α) × new_rep
α=0.9 → slow change → stable reputation
Prevents sudden flip from single bad report
Initialized at 0.95 for all nodes
ALLOW
rep ≥ 0.80
Trusted node. Vote counts with full weight in consensus.
WARN
rep ≥ 0.60
Suspicious. Vote counts at 50% weight. Monitored closely.
QUARANTINE
rep ≥ 0.40
Excluded from voting. Reputation recovery possible.
SLASH
rep < 0.40
Permanently removed. Score → 0. Penalty −0.5 applied.
📡Reputation scores written to blockchain via updateReputation()
🔄Runs every 5 minutes (epoch)
↓
Only ALLOW nodes participate · WARN nodes at 50% weight · QUARANTINE + SLASH excluded
